What happens when you initialize a repository? Why do you need to do it?
You create a new space for different versions. You need to do it so you have a place to keep your files. 

How is the staging area different from the working directory and the repository?
What value do you think it offers?
The staging area is a place to put files that you are working on, before you are ready to make a commit. Also, you can put only the files you want committed there. 

How can you use the staging area to make sure you have one commit per logical
change?
You can compare your file in the staging area to the file in the repository before you commit.

What are some situations when branches would be helpful in keeping your history
organized? How would branches help?
When you want to try something new but don't want to ruin what you have already created. or if you want to make different versions but want to keep a main one.
